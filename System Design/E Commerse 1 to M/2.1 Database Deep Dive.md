# 📘 What Are Databases? (System Design Basics)

## ✅ What is a Database?

A **database** is a system that stores and manages data in a structured way.

You can perform operations like:

* **Create (Add data)**
* **Read (Retrieve data)**
* **Update (Modify data)**
* **Delete (Remove data)**

👉 In simple terms:  
A database is like a **smart storage system** that can efficiently handle data requests.

***

## 🤔 Is a Database a Server?

* A **database itself is software**, but it usually **runs on a server**.
* It listens to requests and responds with data.

✅ So you can think:

* Database = Software managing data
* Database Server = Machine where database runs

***

## ⚖️ Application Server vs Database Server

### ❌ Are they the same?

No, they are **different**.

### ✅ Why keep them separate?

| Application Server      | Database Server           |
| ----------------------- | ------------------------- |
| Contains business logic | Stores data               |
| Easy to restart         | Harder/heavier to restart |
| Handles API requests    | Handles data queries      |

### 🚫 Can we directly connect frontend → database?

* Yes, **technically possible**
* ❌ But **NOT recommended**

### ✅ Best Practice:

Keep **business logic separate** from the database.

***

## 💡 Why Separation is Important?

1. **Easy Maintenance**
   * Restarting application server is simple
   * Restarting DB can be expensive

2. **Flexibility**
   * Example: Switching from **MySQL → PostgreSQL**
   * Your app logic remains the same

3. **Cleaner Architecture**
   * Business logic stays in application layer
   * Database only handles data storage

***

## 🧠 Industry Practice

* Business logic is **usually written in application code (e.g., Node.js, Java, etc.)**
* Not inside the database (like stored procedures)

✅ Exception:

* If the operation is **very database-specific or performance-critical**

***

## 📂 What is a File System?

A **file system** is how an OS stores files on disk.

### 🤔 Is a database just a file system?

✅ Kind of, but not exactly:

| File System                                    | Database                          |
| ---------------------------------------------- | --------------------------------- |
| Stores unstructured data (files, images, docs) | Stores structured data            |
| No strict format                               | Organized (tables, rows, columns) |
| Basic storage                                  | Advanced querying & indexing      |

👉 Example:

* File system: `/photos/image1.jpg`
* Database: `User(id=1, name="Nikhil")`

***

## 🧾 Structured vs Unstructured Data

* **Structured Data (Database)**
  * Organized
  * Has schema (ID, data types)
  * Easy to query

* **Unstructured Data (File System)**
  * No fixed structure
  * Harder to search

***

## ⚡ Can Databases Be in Memory?

### 💡 Question:

Can we store database in memory (RAM)?

### ✅ Answer:

Yes, but depends on requirement.

### ❓ Key Question:

👉 **Does data need to be durable (persist after restart)?**

***

### ❌ In-Memory Databases (RAM)

* Very fast
* But:
  * Data is lost if system restarts

Examples:

* Redis (mostly in-memory)

***

### ✅ Disk-Based Databases

* Data stored on disk
* Slower than memory, but:
  * ✅ Data is safe
  * ✅ Survives restarts

Examples:

* MySQL, PostgreSQL

***

## 🔥 Summary (Super Simple)

* Database = structured data storage system
* Runs on servers but ≠ same as application server
* Always separate:
  * App logic → Application server
  * Data → Database server
* File system = raw storage, DB = smart structured storage
* Memory DB = fast but not durable
* Disk DB = safe and reliable
